tknTypeParser_e gsmParseTkn ( uint8_t const *const  tkn,
uint8_t *  cmd,
uint8_t *  par,
uint16_t  tknLen 
)

AT token parser.

The function takes a pointer to token, two buffers (cmd and par) and the length of the token. In the AT protocol, most tokens have a command part and optional parameters which specify variable input. Not all tokens have parameters, but all of them have a command part.

This function returns the token type plus the command part in cmd and the parameter part (if present) in par. It turns the "raw" token we get from the tokenizer into something usable for the Engine layer.

Parameters
tknPointer to AT token fetched from the serial port tokenizer
cmdAT command buffer
parAT parameter buffer
tknLenLength of the received token
Returns
Returns the type of AT token in an tknTypeParser_e enum, as well as the command and parameter parts in their corresponding buffers
fsmEvent_e gsmProcessTkn ( gsmEngine_t *  engine)

The processToken function checks if there are unread tokens in the token ring buffer. If so, it reads the oldest token, parses the token through the parse function and calls upon the gsmUpdateFsm function with the result. It then passes along the return value of the gsmUpdateFsm call.

fsmEvent_e gsmSendCmd ( gsmEngine_t *  engine,
const uint8_t *  cmdStr 
)

The gsmSendCmd function first parses cmdStr in the same way as we would do with a token received from the GSM module. This is needed since we are going to use the token to update the FSM. We parse cmdStr, make sure that it is a recognized command and write it in the correct format to the serial port. Afterwards, gsmUpdateFsm is called.
